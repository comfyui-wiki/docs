## 路由

服务器定义了一系列的`get`和`post`方法，
这些方法可以在`server.py`中通过搜索`@routes`找到。
当你在web客户端提交一个工作流时，它会被发送到`/prompt`，这将验证提示并将其添加到执行队列中，
返回一个`prompt_id`和`number`（在队列中的位置），或者如果验证失败，则返回`error`和`node_errors`。
提示队列在`execution.py`中定义，也定义了`PromptExecutor`类。 

### 内置路由

`server.py`定义了以下路由：

|路径|get/post|目的|
|-|-|-|
|`/`|get|加载舒适的网页|
|`/embeddings`|get|检索可用的嵌入名称列表|
|`/extensions`|get|检索注册了`WEB_DIRECTORY`的扩展列表|
|`/workflow_templates`|get|检索自定义节点模块和关联模板工作流的映射|
|`/upload/image`|post|上传图片|
|`/upload/mask`|post|上传掩码|
|`/view`|get|查看图片。有很多选项，请见`server.py`中的`@routes.get("/view")`|
|`/view_metadata`/{folder_name}|get|检索模型的元数据|
|`/system_stats`|get|检索系统信息（python版本、设备、VRAM等）|
|`/prompt`|get|检索当前状态|
|`/prompt`|post|将提示提交到队列中|
|`/object_info`|get|检索所有节点类型的详细信息|
|`/object_info/{node_class}`|get|检索一个节点类型的详细信息|
|`/history`|get|检索队列历史记录|
|`/history/{prompt_id}`|get|检索特定提示的队列历史记录|
|`/history`|post|清除历史记录或删除历史记录项|
|`/queue`|get|检索队列的状态|
|`/interrupt`|post|停止当前工作流|
|`/free`|post|通过卸载指定的模型来释放内存|

### 自定义路由

如果你想在执行过程中从客户端向服务器发送消息，你需要向服务器添加一个自定义路由。对于任何复杂的情况，你将需要深入研究[aiohttp框架文档](https://docs.aiohttp.org/），但大多数情况都可以按照以下方式处理：

```Python
from server import PromptServer
from aiohttp import web
routes = PromptServer.instance.routes
@routes.post('/my_new_path')
async def my_function(request):
    the_data = await request.post()
    # the_data现在持有一个字典，包含发送的值
    MyClass.handle_my_message(the_data)
    return web.json_response({})
```

<Tip>除非你知道自己在做什么，不要尝试在类中定义`my_function`。 
`@routes.post`装饰器做了很多工作！相反，定义函数如上所示，然后调用一个类方法。</Tip>

<Tip>你也可以定义一个`@routes.get`，如果你不是改变任何东西。</Tip>

客户端可以使用这个新路由，通过发送一个`FormData`对象，代码类似于这样，这将导致上述代码中的`the_data`包含`message`和`node_id`键：

```Javascript
import { api } from "../../scripts/api.js";
function send_message(node_id, message) {
    const body = new FormData();
    body.append('message',message);
    body.append('node_id', node_id);
    api.fetchApi("/my_new_path", { method: "POST", body, });
}
```
